# harness-designing — product one-pager

Tier 1 context — always loaded. ≤30 lines.

## What it is

A plug-in for design teams building AI harnesses. Ships skills that explain the five-layer framework (Context Engineering / Skill Curation / Workflow Orchestration / Evaluation Design / Knowledge Compounding) and scaffold one in a user's repo. Companion to "Stop Chasing Design Tools. Start Building a Design Harness."

## Who it's for

**5–20 person design teams** adopting AI-assisted workflows. Secondary: solo designers and 20+ person design orgs. Not: non-design teams.

## Core value proposition

Turn scattered AI use (prompts, Figma comments, design-system rules, review habits, thread lessons) into a working five-layer harness so design knowledge **compounds** instead of resetting with each tool release.

## Out of scope (non-goals)

- Not a full design tool replacement (sits around Figma/Claude/etc., not instead of)
- Not a SaaS (local-first, file-based, git-versioned)
- Not a curriculum (the article teaches; plug-in ships a working example)
